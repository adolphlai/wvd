---
description: 自動分析變更並生成規範化的本地 Commit
---

### 第一步：分析暫存區變更
1. 執行 `git diff --cached` 獲取當前已暫存（Staged）的程式碼變動。
2. 如果沒有暫存內容，主動詢問用戶是否要將所有變更加入暫存（`git add .`）。

### 第二步：AI 生成 Commit 訊息
1. 根據程式碼變更內容，判定類型並撰寫訊息，格式要求如下：
   - **新增功能**：`feat: [簡短描述]`
   - **修復問題**：`fix: [簡短描述]`
   - **文檔修改**：`docs: [簡短描述]`
   - **代碼重構**：`refactor: [簡短描述]`
   - **效能優化**：`perf: [簡短描述]`
2. 在標題下方加上詳細的變更描述（Detailed description），說明「為什麼改」以及「改了什麼」。

### 第三步：執行本地 Commit
1. 向用戶展示擬定的 Commit Message。
2. 不必經用戶確認後，可自行執行指令：
   - `git commit -m "[標題]" -m "[詳細描述]"`

### 第四步：完成提醒
1. 告知用戶已成功完成本地提交。
2. **提醒**：此操作僅在本地執行，尚未推送到遠端倉庫。
